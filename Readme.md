# Mini2D Game Engine (C++ / SDL2)

## 项目简介

本项目使用 **C++ + SDL2** 从零实现一个**最小可用的2D游戏引擎（Minimal 2D Game Engine）**，用于学习游戏开发中的核心架构与底层原理。

引擎遵循以下核心模型：

> **Game Engine = 数据（Data） + 更新规则（Update） + 渲染（Render）**


## 项目目标

* 理解游戏引擎的基本运行机制
* 实现一个可扩展的2D引擎基础框架
* 掌握模块化设计与系统解耦
* 最终开发一个简单2D小游戏

## 项目范围

为了控制复杂度，本项目不包含以下内容：

* 3D 渲染
* 复杂物理系统（刚体/力学模拟）
* 脚本系统（Lua / Python）
* 多线程架构
* 高级渲染（光照 / Shader）

## 引擎架构

引擎由多个独立模块组成，每个模块职责单一、相互解耦：

```
Input → Update → Render
```

---

## 核心模块

### Game Loop（游戏主循环）

负责驱动整个引擎运行：

* 处理输入（Input）
* 更新游戏逻辑（Update）
* 渲染画面（Render）

**要求：**

* 支持 delta time（避免帧率依赖）
* 稳定循环运行


### 渲染系统（Rendering System）

功能：

* 清屏（Clear Screen）
* 绘制2D纹理（Texture）
* 支持多个对象绘制


### 输入系统（Input System）

功能：

* 键盘输入（按下 / 松开）
* 鼠标输入（可选）


### 实体系统（Entity System）

功能：

* 管理多个游戏对象
* 支持动态创建 / 删除实体

基础要求：

* 实体包含：

    * 位置（Position）
    * 速度（Velocity）
* 提供统一接口：

    * `update()`
    * `render()`

### 物理系统（Physics System）

功能：

* 基础运动（位置 + 速度）
* 重力模拟
* AABB 碰撞检测

核心机制：

* 使用速度更新位置
* 检测实体之间的碰撞

### 场景系统（Scene System）

功能：

* 管理当前游戏状态
* 支持场景切换（Game / GameOver）

### 资源管理系统（Resource Manager）

功能：

* 加载 PNG 图片（SDL2_image）
* 缓存资源（避免重复加载）


### 音频系统（Audio System）（可选）

功能：

* 加载音频文件
* 播放音效 / 背景音乐

## 技术栈

| 类型     | 技术         |
| ------ | ---------- |
| 语言     | C++        |
| 图形库    | SDL2       |
| 图片加载   | SDL2_image |
| 音频（可选） | SDL2_mixer |


## 最终成果

使用该引擎开发一个简单2D游戏，例如：

* 简单跳跃游戏
* 躲避障碍游戏

**要求：**

* 游戏可运行
* 包含完整游戏循环
* 支持输入、碰撞、得分机制
